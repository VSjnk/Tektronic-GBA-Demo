About Tektronic Wave

This tool is a wave to C/C++ converter/generator mainly designed to use in GBA sound development.

The software is able to convert multiple input to output wave formats. Based on a practical Windows graphical user interface, it is possible to form a list of input waves, set the output desired format (8-44.1 KHz sampling rate, 8 bit, Mono for GBA) for each wave file in list, test the converted format and finally to generate C/C++ module and header files suitable to use in GBA sound development. Most of the PCM formats are supported for input waves files (8/16 bit,Mono/Stereo/8-44.1 KHz).

Requirements: Windows 98/Millenium/NT/2000. Note: DirectX is not required.

Current version: 1.0

Using Tektronic Wave

Tektronic Wave is easy to use and understand only using it for a few minutes. For this reason it will be explained only the main functionality. Below it is shown the main window:

For each wave file added in list you can see the original wave properties. When you add an wave file, it will be asked to associate C/C++ variable. Using "at sampling rate [KHz]", for each wave file in list you mark the output sampling rate, implicitly at 8 bit, Mono (suitable for GBA sound). Before to generate, use "Play converted" button to hear the differencies. The "Test generator" button will inform you if it is possible to convert between all input to output formats for all wave files in list. The wave files must use PCM format. After pressing "Generate" button, the tool will generate two C/C++ files, "sound.c" and "sound.h" in the current directory. Along with Tektronic Wave comes the "TektronicWave.h" file. This file is #include (d) in both above generated files. See it for more details. The data samples are converted as const signed/unsigned char C/C++ array. In your project #include "sound.h" and compile & link the "sound.c" file. In fact, you will use the associated variables as PCMSOUND typedef structure defined in "TektronicWave.h" header file. See source demo for GBA in download section of this help file to see a real use of the associated variables. The PCM data format in wave files have the following values: 0 low - 128 middle - 255 high. For GBA, the samples must be (-128) - 0 - 127. The tool convert between these ranges if you check the "PCM for GBA" option(default). If not, the range values for samples remain unchanged.

See here (local link): go.wav a wave source file with 22.05 KHz sampling rate, 8 bit, Stereo, 2x37800 samples, the generated sound.c file at 8 KHz, 8 bit, Mono, 13744 samples (important: see also the end of file) and the generated sound.h file where the associated variables are defined.

Note: it is possible that the number of samples of input source not to be equal with the number of samples in output (char C array).
